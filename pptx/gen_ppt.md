# 演示文稿生成指南

## 文件隔离
为当前 PPT 任务创建独立的工作目录，并将所有产物写入此目录：
```
output/<ppt-dir>/
├── visual_exploration.md    # 风格探索设计
├── design.md                # 设计文档
├── outline.md               # 大纲文档
├── cover_template/          # 封面模板文件
└── pages/                   # 每页PPT的HTML内容
```

## 工作流程

### 第1步：用户需求和受众分析
进行场景调研，分析用户对最终演示文稿的需求，并写入`<ppt-dir>/visual_exploration.md`，包括：
1. **判断场景类型**：根据用户需求，将 PPT 归入以下 MECE 分类矩阵中的一个格子：
  - **第一维度（阅读方式，决定密度）**：演讲辅助型（低密度、大字、留白） vs 独立阅读型（高密度、数据密集）
  - **第二维度（沟通目的，决定结构）**：告知 / 说服 / 教学 / 展示
2. **封面/结语图片决策**：根据场景类型和主题特征，判断封面页/结语页是否需要生图。
  - **封面/结语页**：适用 / 禁用 — 基于场景类型和主题特征判断是否使用背景图
  - **硬性约束**：
    - 人物介绍、地图类主题的封面/结语背景图**必须**填"禁用"
    - 正式/庄重场景（毕业典礼、政府汇报、颁奖典礼）优先纯 CSS 视觉方案
3. **输出封面/结语图片决策表**：调研完成后，必须将封面/结语图片策略结论填入以下固定格式的决策表，**不得用自由文本代替**：
   ```markdown
   | 决策项 | 结论 | 调研依据 |
   |--------|------|----------|
   | 封面/结语背景图 | 适用 / 禁用 | 调研发现同类PPT... |
   | 图标(FontAwesome) | 适用 / 禁用 | 场景为正式/商业... |
   ```
   **决策表填写规则**：
   - 封面/结语背景图必须填写明确的"适用/禁用"结论，禁止模糊表述（如"可以考虑""视情决定"）。
   - 图片生成前必须先阅读 `reference/image_prompting.md`，再根据页面布局和主体位置要求编写 prompt。

### 第2步：风格探索阶段
基于主题和搜索结果，设计 **2-3 套一眼可区分** 的方案，并将方案追加写入`<ppt-dir>/visual_exploration.md`。
**方案数量决策**：
| 条件 | 方案数量 | 说明 |
|------|---------|------|
| 主题涉及现实中存在**明确品牌色/视觉规范**的企业、机构或品牌 | 2 套 |
| 无明确现实品牌色约束，或主题偏通用/抽象 | 3 套 |
**不同方案说明**：
- 方案 1（常规）：输出一套最符合用户主题的方案风格，如果相关内容现实存在，严格遵循该品牌的真实配色和视觉调性，品牌色需通过搜索获取真实配色。
- 方案 2（差异化探索）：在符合主题的前提下，**必须使用与方案 1 完全不同的主色调**，并改变质感或构图。
- （如有）方案 3：更大胆的视觉尝试，色相和字体气质与前两套完全不同。

**差异化约束（防同质化铁律）**：
1. **背景色必须反转**：如果方案 1 是深色背景，方案 2 必须是浅色背景，方案 3 可以是高饱和度或完全不同色相。**严禁出现两个方案都是同一深色背景的情况**。
2. **色相族必须拉开**：不同方案的主氛围色须落在不同色相区间。禁止用"同一色相 + 微调饱和度"作为新方案。
3. **质感轴**：为每个方案指定不同的画面质感（如"扁平高对比"、"玻璃/折射"、"纸张/印刷噪点"、"摄影暗角"等）。
4. **构图与动势**：在风格句中明确不同的版式动势（如"静态居中栅格"、"对角切割"、"非对称 editorial"、"环形放射"）。
5. **字体气质必须分叉**：展示字体 + 正文字体的组合在不同方案中须呈现不同气质。
6. **质量底线**：非复古主题则必须现代、高冲击；主氛围忌脏灰土色，强调色须亮且饱和。走黑白极简时，用质感或构图补冲击。渐变须服务于已定质感，严禁不同方案使用同一类软渐变。

每个方案须包含：
- **风格描述**：英文短句（< 20 词）附带括号内中文翻译。描述高度独特、视觉效果惊艳的世界级美学设计风格。
- **设计哲学**：
  - **核心原则**：一句话总结这个风格的设计哲学（如"用克制表达力量"、"让数据自己说话"）
  - **情感关键词**：5-8 个形容词定调（如 Authoritative, Refined, Dramatic, Confident, Timeless）
  - **视觉参考**：类比 2-3 个现实世界的设计参考（如"苹果发布会的极简感 + 经济学人的数据密度"）
  - **反面定义**：明确 3-4 条"这个设计不是什么"（如"不是可爱的、不是花哨的、不是渐变堆砌的"）
- **调色板**（完整 Design Token，不只是色号列表）：
  - `background`：页面主背景色（HEX）
  - `foreground`：主文字色（HEX）
  - `muted`：次要背景色/卡片底色（HEX）
  - `accent`：强调色（HEX）——用于关键数字、标题高亮、核心数据点
  - `border`：边框/分割线色（HEX）
  - **配色必须经典、高级且一致**，推荐采用"深色主调 + 亮色强调"（如深蓝+金色、深灰+亮橙）或"纯净浅色 + 沉稳主色"的组合，避免使用廉价的渐变或不协调的高饱和度撞色。**主色+锐利强调色 优于 胆怯的均匀分布色板。**
- **字体方案**：展示字体（大标题用）+ 正文字体（正文用）的搭配方案，仅允许以下字体：
  - `Microsoft YaHei` 
  - `SimHei`
  - `SimSun`
  - `KaiTi`
  - `FangSong`
  - `幼圆`
  - `隶书`
- **Bold Choices**（3-5 个非协商的设计亮点，让设计有个性而非千篇一律）：
  - 每个方案必须列出 3-5 个具体的、可执行的视觉承诺（如"封面标题用 72px 衬线体"、"每个 section 过渡页用全黑反色"、"数据页用渐变色条而非普通柱状图"、"所有卡片使用 1px 细线边框而非阴影"）
  - Bold Choices 是该方案的视觉签名，HTML生成阶段必须逐条兑现

### 第3步： 生成风格封面页（HTML）
阅读 **`skills/pptx/html_slide_technical_design.md`** 文件，生成封面页 HTML 时，遵循里面的 HTML 技术规范。
根据第3步设计方案交替输出：先输出风格描述文字 -> 在cover_template/目录下写入该方案的封面 HTML 文件如`slide_cover_01.html` -> 完成后再做下一套。
单个方案的输出流程：
1. 输出文字，包含：`方案 {N}：{英文风格描述} （{中文风格描述}）`、调色板、字体搭配。
2. 如封面图片决策表中"封面/结语背景图"为"适用"，必须先阅读 `reference/image_prompting.md`，然后调用 `generate_image` 获取图片 URL。
   - prompt 不得只写"留白给文字"，必须用光线、纵深、雾化、暗部/亮部平面、主体边缘化等画面语言形成安全区。
   - HTML 中必须直接引用 `generate_image` 返回的 URL，禁止引用本地路径。
3. 使用 `file(brief=..., action=write, path=..., content=...)` 工具写入完整 HTML。
   - **文字安全区**：根据生图描述中的主体位置描述明确封面文字所在的安全区域，并预留至少一定内边距
   - **图片主体区**：若使用背景图，主体区必须与文字安全区错开；主体、亮部、高对比细节不得进入文字安全区
   - **封面内容铁律**：封面页只能包含：主标题、副标题、演讲者/公司信息、日期。**严禁在封面页放置任何详细数据（如营收、利润、百分比）、图表或长段落文本**。封面是定调的，数据是内容的，绝不能混淆。
4. 完成后再进入下一个方案。

### 第4步：用户选择
所有封面 HTML 全部生成后，截图展示并总结：
```bash
bash "$SKILL_PATH_BASH/pptx/scripts/pptx.sh" screenshot \
  /path/to/cover_1.html \
  /path/to/cover_2.html \
  /path/to/cover_3.html
```
将几个方案的**截图作为附件发送给用户，再使用`ask_user_question工具`询问用户**选择哪个方案，并同步确认是否配图。配图问题须使用以下文案与选项（严格按此固定顺序，选项 1 为"是"，选项 2 为"否"）：
**PPT 是否需要配图？（配图会消耗更多灵点）**
- **是**，适当配图增强表达
- **否**，不配图
用户选"是"时，后续可在需要时使用搜图或生图，仍须遵循 `gen_html_ppt.md` 中的配图策略与生图约束，无图优先、仅在图片确有信息功能时使用。用户选"否"时，整套 PPT 均不使用搜图或生图。
用户选定后，再阅读`skills/pptx/gen_html_ppt.md`文件进入正式生成html ppt阶段
**如果用户对所有方案都不满意，应从下一个序号（如4、5、6）开始新设计一套方案，而不是复用或覆盖原有的设计。这样可以确保每个设计方案都被完整保留，让用户能够比较和回溯所有已生成的选项，避免有价值的信息被覆盖或丢失。**
