## HTML PPT 逐页生成指南

### 第1步：视觉设计
根据用户选定的风格，进行视觉设计。
`design/` 目录提供按"沟通目的"划分的风格预设。先判断本次演示的主导目的，再读取对应文件。**以下所有维度均基于参考来源的默认指导；用户的明确要求始终优先。**
| 文件 | 主导目的 | 覆盖场景（含场景适配小节） |
|---|---|---|
| `design/inform.md` | 告知 / 汇报 | 工作汇报、项目/季度总结、绩效述职、政务与国企报告、行业研报的事实呈现、学术与科研报告/答辩 |
| `design/persuade.md` | 说服 / 提案 | 项目提案、商业计划书、融资路演、创业大赛、投资建议、战略规划、竞品与市场的观点驱动分析、内部立项、公益倡导/主题宣传 |
| `design/teach.md` | 教学 / 培训 | 中小学/高校课件、公开课与讲座、学生课堂演示、企业培训（入职/安全/技能）、教程与操作指南、文化介绍/科普 |
| `design/showcase.md` | 展示 / 推广 | 品牌发布、新品介绍、营销活动、艺术/时尚展示、主旨演讲（keynote） |
| `design/general.md` | 通用 | 用途不明确，或夹杂多种目的的混合场景 |

- 主导目的明确时，直接读取对应文件
- 介于多个目的之间时，选最主要的目的作为主预设，可借用其他预设的个别技巧
- 各文件内的"## 场景适配"小节给出细分场景（如政务报告、企业培训、融资路演等）的额外要求
在阅读完参考后，结合用户选择的**风格方案**，输出`design.md` 文件
`design.md` 必须包含以下内容（按章节顺序写入）：
- **1. 内容分析与叙事设计**：受众、情绪、核心论点、叙事框架（如金字塔/SCQA）与节奏设计
- **2. 视觉设计落地**：直接继承用户选定方案的风格句、设计哲学，并定义一个贯穿全套的视觉母题（如圆角框架、粗体单侧边框），并根据场景（教育/商业/产品/技术）给出具体的视觉语言落地
- **3. Design Token 表**：将用户选定方案的调色板，字体方案展开为完整的 CSS 变量表，作为后续生成html ppt页的唯一色彩来源。除非用户指定，否则**任何正文字体不小于22px**
- **4. Bold Choices**：直接复制用户选定方案的 3-5 个 Bold Choices（非协商的视觉承诺），生成内容页html时必须逐条兑现

- 注意：**视觉设计应以内容为基础——内容少并不意味着设计可以简单；反而要求更高端、更精致的美感！** 内容越少，设计越需要提升精致感。如果内容偏稀疏，或每页文字量较少，就需要更多装饰元素、图片和更精细的网格布局来确保高端的视觉体验。

### 第2步：内容大纲设计
根据视觉设计中的信息规范，输出整份ppt的大纲 `outline.md` 文件：

1. 大纲第一部分，整体规划，包含以下内容
- **内容分析与叙事设计**：
基于 `design.md` 里这部分内容再微调，要考虑以下几点
明确核心：受众是谁？场景是什么？核心情绪（权威/激情/克制）？唯一要记住的结论是什么？
叙事结构：如何开场，如何展开，如何收尾。MECE 原则检验：确保各分论点之间相互独立、完全穷尽。
节奏设计：哪里需要高密度信息，哪里需要留白呼吸，哪里需要视觉冲击。
- **场景感知**：
  - **教育/培训**：强调清晰的层级、步骤图、对比图，减少干扰元素。
  - **商业/汇报**：强调数据可视化、金字塔结构、结论先行，配色沉稳。
  - **产品/发布**：强调大图展示、特性网格、极简大字，视觉冲击力强。
  - **技术/架构**：强调系统图、代码块高亮、逻辑流程，配色科技感。
- **页数规划**：
根据用户提供的文档或搜索资料的文本量，确定合适的幻灯片总数。内容要丰富详实，不要因为压缩内容而损失信息密度：

| 提供文本量 | 建议页数 | 对应篇幅 |
|-----------|---------|---------|
| < 8000 字 | 15–20 页 | 短篇幅 |
| 8000–12000 字 | 25–30 页 | 中篇幅 |
| > 12000 字 | 35–40 页 | 长篇幅 |

若用户未提供具体资料，根据主题复杂度和受众预期自行判断，但不得少于 12 页。
在写入大纲文件时，先确定合适的 PPT 篇幅，再开始写入大纲内容。

> **注意**：确定篇幅时，向用户说明的是一个页数范围（如"约 25–30 页"），而不是一个精确数字。实际页数会在大纲完成后自然确定，过早锁死页数反而会导致内容被强行填充或裁剪。

2. 大纲第二部分，逐页编排内容大纲，遵循以下要求规划每页大纲

- **结构完整性铁律**：
  - 只要 PPT 存在“目录页”，就必须存在与目录中各一级章节一一对应的“章节页”。
  - **第一章不得例外**：目录页之后，必须先进入“第一章章节页”，再进入第一章的第一张内容页。
  - 章节页不是可选装饰页，而是叙事结构的硬性组成部分，用于完成“目录导航 -> 章节开场 -> 章节内容”的过渡。
  - 若全篇只有 1 个一级章节，则可以不单独设置目录页；但如果已经设置了目录页，仍必须保留该唯一章节的章节页。
  - 若篇幅极短（总页数 <= 8）且章节本身只有 1 个，可采用“目录与章节合并页”；除此之外，目录页与章节页必须分开。
- **内容量评估铁律**：规划每页内容时，必须评估每页的核心内容量是否足以撑满一页。
  - **研报场景单页内容量底线**：对于行业研报、白皮书等高密度场景，单页大纲必须包含以下组合之一，严禁出现内容单薄的页面：
  组合 A：1 个主图表（ECharts） + 1 段完整的分析结论（50-100字） + 3-4 个辅助数据卡片
  组合 B：2-3 个并排的小图表 + 对应的分析要点
  组合 C：1 个多行多列的复杂数据表格 + 底部总结说明
  如果原始素材内容不足以支撑上述组合，必须通过逻辑推演补充合理的占位数据/分析，或将多个单薄的页面合并为一页。
  - **演讲场景**：如果一页的内容只有 2-3 个要点且无法扩展，**必须与相邻页合并**，或者补充相关图表/对比数据，绝不允许出现内容单薄导致大面积留白的页面。
- **布局与空间分配原则**：每页 HTML 的空间结构遵循**Grid 定骨架，Flex 分比例**：
  - **Grid** 负责将 slide-container 切成垂直分区（标题带 / 主内容区 / 页脚带），并在主内容区内网格编排子内容块，确保整页视觉对齐、内容区块之间不重叠且合理占据 1280×720px 画布。
  - **Flex** 负责在子内容块内按比例分配空间，子内容区块多用flex流式布局，多列用 `flex:1` 等分宽度，多行用 `justify-content:space-evenly` 或 `flex:1` 撑满高度。总之，flex的核心目的是让每个子内容区块也重心居中，并减少大量留白空间。
  - **禁忌**：主内容区没有明确空间分配指令时，子项默认堆顶、下方空洞——这是最常见的视觉重心偏移的问题。
- **布局与样式丰富性**：整份 PPT 须有意识地在各页之间制造视觉变化，避免所有页面"同一个模子"带来的审美疲劳。整体性靠统一的配色/字体/视觉母题保持，变化靠布局和样式细节实现：
  - **布局变化**：内容区的分栏数、主从比例、横纵方向的切割方式，应随内容结构灵活调整
  - **样式变化**：卡片有边框 vs 无边框、标题加粗 vs 细线分割、装饰元素密 vs 疏——各页在样式细节上有节奏地交替，而非每页都用同一套视觉处理
- **配图与布局一体规划**：逐页大纲必须把配图作为布局的一部分，并在本阶段逐页决定内容页是否配图。
  - 需要配图的页面，必须说明图片数量、每张图的信息功能、画面主体位置、页面占比和与文字/图表的关系。
  - 支持单图、多图并列、多图序列、主图+辅图、背景图+前景内容等布局。多图页面必须说明每张图的尺寸比例和网格位置，避免多张图互相抢焦点。
  - 图像 prompt 中的主体位置必须与页面布局一致。例如右侧配图就要求主体偏右并在左侧留出自然空场；三图并列就要求三张图的视角、尺度或内容职责彼此区分。
  - 数据页、表格页、流程页优先使用图表、表格、CSS 结构图或图标；只有当图片能承担明确的信息功能时才生图。
- **内容页配图原则**：
在 `outline.md` 阶段根据场景调研结果，结合每页的核心内容、信息功能、事实属性和布局空间决定是否配图。
  - 真实人物介绍、地图类主题的内容页优先纯 CSS 视觉方案
  - 正式/庄重场景（毕业典礼、政府汇报、颁奖典礼）优先纯 CSS 视觉方案
  - 生图必须服务明确的信息功能，并在布局中有稳定位置；不能为了填空而配图
- **逐页大纲格式**：每页大纲必须包含以下字段：

```
- 类型：封面页/目录页/章节页/内容页/结语页 其中之一
- 视觉焦点：用简洁明了的一句话突出本页的核心视觉或主要信息，让阅读者一眼即可把握本页要点。
- 风格关联：深色背景 + 强调色装饰线
- 核心内容：(简述该页要传达的核心信息)
- 配图：生图 / 真实素材 / 无
- 图片规划：
  - 图片数量：0 / 1 / 2 / 3 / ...
  - 布局位置：说明每张图在页面 grid/flex 中的位置、占比、裁切比例和留白方向
  - 信息功能：说明每张图服务什么内容判断
  - 画面提示：仅描述画面主体、构图、位置、尺度、光线和禁忌；正式生图 prompt 在生成图片前再根据提示词最佳实践补全
- 布局规划：
  - 内容层级：分析本页各内容元素的逻辑关系（并列/从属/对比/递进）
  - 布局方案：根据内容层级自由设计，说明分栏结构和主从比例
  - 页面层：slide-container grid 轨道 -> 枚举每个轨道对应的直接子元素
  - 主内容层：主内容区 grid/flex 轨道 -> 枚举每个子内容块和所有视觉子元素（含分隔符等中间元素）
- 字体方案：主标题 XXpx / 副标题 XXpx / 正文 XXpx / 辅助 XXpx，需满足以下要求：
内容块<=2个:正文>=32px；
内容块=3个:正文>=28px；
内容块=4个:正文>=26px；
内容块>=5个:正文>=22px
```

**逐页大纲各字段说明**：
- **布局规划·内容层级**：**写布局前先分析内容的逻辑关系**——什么是并列关系（同级网格）、什么是从属关系（子块嵌套在父块内）、什么是独立分区（应用分隔线或 gap 隔开）。布局从内容语义出发，自由设计，不要套固定模板。
- **布局规划·布局方案**：根据内容层级自行决定分栏结构，同时有意与相邻页拉开差异。
- **布局规划·页面层/主内容层**：枚举所有视觉子元素并对应轨道，元素数与轨道数必须一致；写 HTML 时直接按此执行。
- **配图**：在 `outline.md` 阶段逐页独立决策，禁止直接引用风格探索阶段的图片决策表。必须根据本页内容判断使用生图、真实素材或无。配图必须与布局规划一致，不能脱离页面结构临时插入。
- **图片规划**：当图片数量大于 0 时必须填写完整。多图页面必须逐张说明角色和位置；背景图必须说明前景文字安全区；如果不生图则写"不适用"。
- **字体方案**：列出本页各层级具体字号，标注内容块数量以确认满足字号自适应规则。

### 第3步：再次读取HTML 技术规范文档
**必须执行**：再次读取 **`skills/pptx/html_slide_technical_design.md`** 文件（即使前面已经读过了），接下来生成的每页html需严格遵循其中的 HTML 技术规范与设计指导。

### 第4步：逐页生成PPT的HTML
如果在风格探索阶段已经生成了封面 HTML，无需重新生成，直接复用原有文件。请将封面页及其所用图片（如有）一并复制到 `pages/` 目录，并将封面文件命名为 `slide_01.html`。**请使用 Python 脚本实现自动拷贝操作**

**写入方式**：使用工具 `file(brief=..., action=write, path=..., content=...)` 将完整 HTML写入`slide_xxx.html`文件。**禁止**使用 Python 脚本写入 HTML 文件。
**编码注意**：避免在代码、注释、字符串中使用特殊 Unicode 字符，使用标准 ASCII 或常见中文字符。

#### 单页实现节奏

1. 按`outline.md`文件中各页的配图和图片规划判断是否需要生图。
   - 如果本页需要生图，必须先阅读 `reference/image_prompting.md`，再根据其中的提示词最佳实践编写 prompt。
   - 每张图单独编写 prompt；多图页面必须让每张图的主体、构图、尺度或视角承担不同职责，避免生成一组重复图片。
   - prompt 必须继承 `outline.md` 的布局位置，并包含禁止文字的约束。
   - 获取生成图片 URL 后，直接在本页 HTML 中**引用该 URL**。
2. 根据各页的布局规划、图片规划、风格关联、视觉焦点、核心内容，思考 HTML 要填充的内容
3. 兑现 Bold Choices：检查这页是否有机会兑现某个 Bold Choice
4. **布局自检**：对照 `html_slide_technical_design.md` 的 Grid + Flex 布局原则，在写入 HTML 前逐条确认：
   - **Grid 轨道逐个对应**：每个 grid 容器，其**所有**直接子元素（内容块 + 箭头 + 分隔线等装饰元素）与轨道数必须总数一致
   - **主内容区垂直居中**：`.slide-main` 等 `1fr` 容器有 `display:grid; align-content:center`；内部子块轨道全用 `auto`，禁止混用 `auto 1fr`（`1fr` 会独吞剩余空间导致底部空洞）
   - **字号与内容量匹配**：内容块数量对应字号档位
   - **图片与文字安全区**：图片主体不能压住标题、正文、图表和页脚；背景图必须有明确的前景安全区；多图布局必须有稳定间距和统一裁切规则
   - **视觉重心**：脑中三等分页面，内容重心落在中间 1/3
   - **内容少时有装饰补充**：内容块 ≤ 3 个时须有装饰元素填充画幅
5. 写入这页的html文件

### 第5步：跑版校验

所有幻灯片 HTML 文件生成完毕后，调用 `check_slides_layout` 对全部文件进行**HTML 合法性**和**跑版**检测。

#### 执行方式：

```bash
bash "$SKILL_PATH_BASH/pptx/scripts/pptx.sh" check-layout \
  /path/to/slide_01.html \
  /path/to/slide_02.html
```

---

### 第6步：导出 PPTX

所有幻灯片 HTML 文件校验通过后，导出 PPTX：

```bash
bash "$SKILL_PATH_BASH/pptx/scripts/pptx.sh" html-to-pptx \
  /path/to/slide_01.html \
  /path/to/slide_02.html \
  -o /path/to/output.pptx
```

## 最终强调

| 原则 | 含义 |
|------|------|
| **严格按步骤执行，禁止跳步** | 必须完成"视觉设计 -> 内容大纲"两步后，再进入逐页 HTML 生成。禁止在大纲未完成时提前动手写 HTML，否则布局和风格无从对齐 |
| **布局是第一优先级** | 每页 HTML 开始写入前，先想好这页的布局是否符合要求，视觉重心有没有居中，整页留白是否过多，各内容区块间有没有对齐 |
| **布局从内容语义出发** | 布局结构必须反映内容的逻辑关系，禁止用固定模板套内容、强行将逻辑上不对等的内容放进等分列 |
| **布局与样式须有变化** | 整份 PPT 的整体性靠统一配色/字体/视觉母题保持，变化靠布局骨架和样式细节实现 |
| **字号随内容量浮动，不得死用最小值** | 内容块 <=4 个时，正文字号须按自适应规则上调；用字号和间距撑满空间，而不是用留白将就 |
