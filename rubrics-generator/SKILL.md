# Rubrics 评分标准生成器

根据用户提供的课程信息，自动生成一份多维度的 Rubrics 评分标准，并可导出为格式化 Word 文档。

## 工作流

### 1. 收集课程信息

向用户确认以下信息（允许部分缺失，缺失项根据任务合理推断）：

| 信息项 | 必要性 | 说明 |
|-------|-------|------|
| 课程/学科名称 | 推荐 | 用于确定学科适配模式 |
| 教学主题/课题 | 必要 | Rubrics 标题的核心内容 |
| 教学目标 | 推荐 | 推导评价维度的主要依据 |
| 教学对象 | 推荐 | 影响描述语的语言难度 |
| 任务类型 | 必要 | 技能练习/小组合作/展示/比赛等 |
| 评价重点 | 可选 | 教师特别关注的方面 |
| 权重分配 | 可选 | 不提供则默认等权重 |

如果用户已提供足够信息，直接进入第 2 步，无需重复询问。

### 2. 确定评价维度

- **通用学科**：从教学目标提取 4-5 个核心维度（如知识理解、技能掌握、合作交流等）
- **体育课**：读取 `references/pe-dimensions.md`，根据课型和运动项目选择合适的维度组合

维度要求：
- 覆盖教学目标的不同层面
- 可观察、可衡量
- 维度之间不重叠

### 3. 生成 Rubrics 表格

按照 `references/rubrics-prompt.md` 中的提示词模板生成 Rubrics 内容，包括：

1. **4 个表现等级**：优秀（4）、良好（3）、合格（2）、需努力（1）
2. **每个维度的描述语**：
   - 使用行为动词开头
   - 描述具体可观察的行为
   - 相邻等级有明显梯度差异
   - 语言适合目标学段
3. **权重分配**：如用户指定则按指定值，否则默认等权重
4. **评分说明**：计分方式和达标标准
5. **使用建议**：简要操作建议

### 4. 输出

先向用户展示 Markdown 格式的 Rubrics 表格供审阅。

用户确认后，按以下方式导出 Word 文档：

```bash
RUBRICS_MD="$(cat << 'EOF'
[这里放入完整的 Rubrics Markdown 内容]
EOF
)" OUTPUT_PATH="[输出路径].docx" NODE_PATH=/usr/local/lib/node_modules node scripts/rubrics_docx.js
```

- 脚本路径：`scripts/rubrics_docx.js`
- 必须设置 `NODE_PATH=/usr/local/lib/node_modules`
- `OUTPUT_PATH` 默认为 `/home/lingxi/workspace/Rubrics评分标准.docx`

文档格式特点：
- 深蓝色表头，四级表现等级用绿/黄/橙/红色浅底区分
- 隔行交替背景色，提升可读性
- 包含评分说明和使用建议段落

## 注意事项

- 描述语必须具体可操作，避免"表现很好""基本掌握"等模糊表述
- 体育课的 Rubrics 应体现动作技能维度的特殊性（动作规范性、连贯性等）
- 权重总和必须为 100%
- 如用户对生成的维度或描述不满意，应迭代修改直到满意
