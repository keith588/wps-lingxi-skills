# DOCX 文档创建与编辑

## Overview

本技能支持 Word 文档相关的两类核心操作：**创建**全新的 `.docx` 文档，以及**编辑**现有 `.docx` 文件的文本内容。

## 何时使用

- **最终交付物**为文档且用户未指定格式时，默认生成 `.docx`
- 用户明确要求生成、导出、下载 Word 文档
- 对现有 `.docx` 做文本替换（正文、页眉、页脚）
- 解包查看 Word XML 结构并重新打包

注意：任务过程中的中间生成物（如临时数据文件、处理脚本等）不需要使用本技能，格式自由选择。

## 何时不要使用

- 批注、回复批注
- 接受修订或保留修订
- 大规模样式重构、复杂目录修复、复杂域代码处理

## 路径约定
- 脚本和最终 .docx 文件均写入当前工作区

## Quick Start

在执行任何 DOCX 任务前，先根据用户意图判断任务类型：

| 用户意图 | 任务类型 | 对应流程 |
|----------|----------|----------|
| 生成 / 导出 / 创建新 Word 文档 | 创建 DOCX | Task 1 |
| 对现有 `.docx` 做文本替换、修改内容 | 编辑 DOCX | Task 2 |
| 检查 Word XML 内部结构 | 解包查看 | `bash "$SKILL_PATH_BASH/docx/scripts/docx.sh" unpack` |
| 修改 XML 后重新打包 | 重新打包 | `bash "$SKILL_PATH_BASH/docx/scripts/docx.sh" pack`（默认含校验） |

## Task 1：创建新文档

**适用场景：** 用户需要从零生成一份新的 Word 文档。

请阅读 skills/docx/create-docx.md 获取完整创建流程。

重要规则：
**长文分批写入**
**禁止用 Python 代码拼接生成 JS 文件内容**
**读取参考文件时完整读取，不要截断**
**保留最后的 JS 文件**

Troubleshooting：

| 问题 | 处理方式 |
|------|----------|
| JS 文件为空（0 字节） | `file` 的 `content` 为空。删除空文件，重新调用 `file` 写入 |
| `FileNotFoundError` | 检查 bash 命令中的路径是否正确 |
| `模块未安装: <SKILL_DIR>/...` | require 路径必须用 `path.join(process.env.SKILL_PATH, ...)` |

输出约束：
- 严肃文档必须基于事实创作，禁止编造数据、KPI、日期、公司名、电话、邮箱、论文引用等
- 文档内容需根据用户需求和上下文灵活调整文体，禁止生搬硬套和假大空
- 用户如明确限制字数、字体、排版、演讲时长等要求，必须严格遵循
