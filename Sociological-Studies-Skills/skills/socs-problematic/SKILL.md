# 社会学问题意识（socs-problematic）

## 触发时机

- 问题写成"X 对 Y 的影响"，没有社会学的"为什么重要"
- 田野/访谈材料很丰富，但讲不出"这到底是关于什么的问题"
- 自评："看起来是个发现，但不像个社会学问题"

## 什么是社会学问题意识

不是"两个变量有没有关系"，而是**关于社会机制、结构或过程的困惑**：某种社会安排如何被生产、维持、再生产或瓦解；某种不平等/秩序/认同如何形成。好的问题意识让读者觉得"这背后有更大的社会学关切"。

## 三步把"发现"抬升为"问题"

1. **从结果到机制**：不止问"是否相关"，问"经由什么社会过程"
2. **从个体到结构**：把个体行为放回关系、制度、文化结构中
3. **从特例到理论关切**：这个案例/相关在挑战或印证哪条社会学判断

## 问题意识陈述模板

```
既有研究多把 <现象> 理解为 <既有解释A>。但 <经验线索/反常> 提示，
<现象> 可能源于 <社会机制M>。本文追问：<机制M 如何在 何种结构条件下
生产/再生产 何种社会后果>？这一追问关涉 <社会学关切：分层/秩序/认同…>。
```

## 定量稿与定性稿的问题意识差异

- **定量**：问题意识在"机制与异质性的社会学含义"，不是"识别更干净"
- **定性**：问题意识常在"过程与意义"，在田野中被不断重述、收窄
- 两者都要避免把问题降格为"测一个效应"

## 本刊"问题意识"的高频退稿信号（自查）

《社会学研究》的初审与外审最常因"问题意识不足"退稿。下列信号命中两条以上，多半未立住：

- 引言通篇时代背景，读到第三段还不知道要追问什么。
- 问题可一句话改写为"X 对 Y 有没有影响"，且改写后毫无损失。
- 删掉"政策""对策"二字，问题就不成立。
- 困惑来自"数据里恰好有这两个变量"，而非社会学关切。
- 大词（现代性、治理、转型）反复出现，却落不到一个具体的机制困惑。

## 微型走查示例（数字社会·外卖骑手，数字为示意）

设作者手握一份骑手访谈 + 平台数据，初稿问题写作"算法对骑手劳动强度的影响"。按三步抬升：

```
原始（变量关系，未立住）：
  "本文研究平台算法对外卖骑手劳动强度的影响。" —— 是题目，不是问题。

第1步 从结果到机制：
  不只问"算法是否提高了强度"，问"算法经由什么社会过程组织起骑手的劳动"。
第2步 从个体到结构：
  把骑手的"自愿多接单"放回平台规则、评分系统、用工关系的结构中看。
第3步 从特例到理论关切：
  这挑战了"劳动控制需要科层化监督"的经典判断——算法实现了无监工的控制。

立住的问题意识（一句话）：
  平台算法如何在"自由接单"的外观下，把劳动控制嵌入计价、派单与评分规则，
  使骑手在自我加压中再生产出一种"无须监工的劳动秩序"？
社会学关切：劳动过程中的控制与同意（Burawoy 脉络）+ 数字时代的组织形态。
困惑/反常：没有工头，却有比工头更密的控制；骑手抱怨却又"主动"加速。
机制猜想：算法把控制规则化、游戏化，诱导"赶工同意"的再生产。
路径：混合（访谈呈现意义建构 + 平台数据刻画规则结构）。
```

删掉任何"政策建议"，这个问题依然成立且更清晰 → 立住。

## 自检清单

- [ ] 问题是关于**社会机制/结构/过程**，不是变量净效应
- [ ] 能说清"这关涉哪一类社会学关切"
- [ ] 有一个让人想读下去的"困惑/反常"
- [ ] 问题大小适中：不琐碎，也不空泛到无法经验回答
- [ ] 删去政策/对策，问题依然成立

## 反模式

- "本文研究 A 与 B 的关系"——这是题目不是问题意识
- 用宏大词（"现代性""治理")包装，却无具体机制困惑
- 问题随数据可得性而定，而非由社会学关切驱动

## 输出格式

```
【一句话问题意识】<…>
【社会学关切】分层 / 秩序 / 认同 / 再生产 / …
【困惑或反常】<…>
【机制猜想】<社会过程M>
【路径】定量 / 定性 / 混合
【下一步】socs-theory-dialogue
```
