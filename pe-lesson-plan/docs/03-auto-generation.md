# 教学分析区自动生成

> 本文档说明教学策略文本、策略图、流程图、教学反思的自动生成逻辑。

## 教学分析区的自动生成字段

教学分析区共有七个字段，其中两个具备**自动生成能力**，是本技能的内置环节：

| 字段 | 自动生成 | 规范文件 | 输出 |
|------|---------|---------|------|
| 教学策略 | ✓ | `scripts/strategy_prompt.md` | 精简三段式文本（200-300 字）+ 可视化图示 |
| 教学流程图 | ✓ | `scripts/flowchart_prompt.md` | 动态N列分层流程图 PNG（@ 200 DPI，3~9任务自适应） |
| 教学反思 | ✓ | `scripts/reflection_prompt.md` | 三段式 Markdown 文本（不含顶部标题） |
其余字段（课题名称、教学内容分析、学情分析、知识点/能力点/素质点、教学重难点、教学流程图、课前、教学方法与手段）由用户输入或基于用户输入整理。

### 教学策略

教学策略是教案"教学分析"区的核心字段，决定了整份教案的学术专业度。

#### 输入变量

调用提示词前必须收集以下 6 个变量：

| 变量 | 说明 | 示例 |
|------|------|------|
| 课程性质 | 决定整体策略取向 | 公共体育课 / 体育教育专业课 / 专项课 |
| 教学内容 | 决定项目特异性 | 篮球传切配合 / 跨栏跑 / 少年拳 |
| 授课对象 | 决定差异化设计 | 大二女生 / 体育专业 / 零基础 |
| 课时类型 | 决定理论选用 | 新授课 / 复习课 / 考核课 / 综合课 |
| 教学目标 | 收束对齐 | 运动能力 / 健康行为 / 体育品德 |
| 重点难点 | 支撑策略落地 | 重点—动作路线 / 难点—连贯节奏 |

#### 生成流程

1. **生成文本**：调用 `scripts/strategy_prompt.md`，将 7 个变量（含课程进度上下文）填入占位符，输出**精简三段式**教学策略文本（整体策略取向 / 关键策略组合 / 核心教学目标），**字数严格控制在 200-300 字**（单元格适配）。
2. **生成图示**（仅 AI 生图）：
   - **仅 AI 生图**：调用 `strategy_ai_generator.get_strategy_ai_prompt()` 获取英文提示词，通过 `generate_image` 工具生成定制化策略图（纵向 2:3 宽高比，5.0×10.0 英寸，exact 14400 DXA 拉满整页）。提示词根据运动项目、课时类型、策略文本、理论依据、课题名称、授课对象自动构建。
   - **方案 B（PIL fallback）**：若 AI 生图不可用，调用 `scripts/strategy_visualizer.py` 的 `generate_from_template()` 或 `generate_strategy_diagram()` 生成策略图（matplotlib 绘制）。
   - **兜底**：两者均失败时，`strategyImg` 留空，模板自动使用 `placeholder_strategy.png`（占位符）。
   - 模板自动绑定 5.5 英寸行高（7920 DXA, atLeast），确保图与单元格完全对齐。
3. **质量自检**：按提示词末尾的撰写完成自检清单逐项核对（见下文）。

#### 撰写规范

- **教学策略 ≠ 教学方法**：禁止简单罗列"X 法、Y 法、Z 法"，必须体现对方法手段的整体谋划。
- **项目特异性强制要求**：
  - 球类项目：必须涉及战术决策、空间感知、攻防转换
  - 田径项目：必须涉及能量代谢、动作节奏、技术经济性
  - 体操/武术项目：必须涉及动作迁移、保护与帮助、分解与整合
  - 游泳项目：必须涉及水感培养、呼吸节奏、安全保障
- **理论选用**："自然嵌入"而非"贴标签堆砌"，单段策略引用 1 个核心理论即可。
- **理论-课型映射**：

  | 课型特征 | 优先理论 |
  |----------|----------|
  | 新授课（技能初学） | Fitts & Posner 动作技能学习三阶段 + 程序教学法 |
  | 复习课（技能巩固） | 布鲁姆掌握学习理论 + 变式练习 |
  | 球类项目（战术决策） | TGfU 领会教学法 + Siedentop 运动教育模式 |
  | 复杂技能（体操/武术） | 动作学习阶段性 + 分层教学理论 |
  | 公共体育（兴趣导向） | 加德纳多元智能 + 运动教育模式 |
  | 专项课（高水平） | 周期化训练理论 + Metzler 教学风格谱系 |
  | 国内课程标准导向 | 季浏"核心素养" / 毛振明"三基教学" / 领会教学法本土化 |

#### 反例警示

| 反例类型 | 错误表述 | 问题诊断 |
|---------|---------|---------|
| 方法罗列型 | "本课采用讲解示范法、分解练习法、完整练习法、游戏法……" | 教学方法清单，缺整体谋划与价值取向 |
| 泛化口号型 | "以学生为中心，注重因材施教，培养终身体育意识。" | 放之四海而皆准，无项目特异性 |
| 理论贴标签型 | "依据 Fitts & Posner、TPACK、建构主义、多元智能……" | 理论堆砌，缺内化与情境化 |

#### 质量自检清单

生成后逐项核对：

- [ ] 是否避免了方法罗列（X 法、Y 法、Z 法）？
- [ ] 是否体现了教学内容动作/战术结构分析？
- [ ] 理论引用是否自然嵌入而非贴标签？（单段 ≤ 1 个）
- [ ] 三段（准备/基本/结束）是否有清晰的衔接逻辑？
- [ ] 差异化设计是否具体到可观察的行为？
- [ ] 是否包含项目特异性要求的对应内容？
- [ ] **字数是否严格在 200-300 字之间？（超过 300 字视为不合格）**

#### 教学策略图示

策略文本配套生成可视化图示，**用于教案"教学策略"行下方的视觉化补充**。

**模板行高绑定**（v3.6+）：模板 `blank_template.js` 在生成"教学策略"行时显式设置 `trHeight = 7920 DXA = 5.5 英寸`，`hRule = atLeast`（v4.4.1 修复）。这意味着：

- 4.64×6.97 英寸纵向图（2:3 宽高比），高度拉满整页（10.0 英寸）
- 策略图以 2:3 宽高比直接生成，docx 渲染时按 `col 2-8` 列宽等比缩放，高度受 exact 14400 DXA 行高（拉满整页）约束
- 任何外部传入的策略图都会按 5.0 英寸宽度自动等比缩放，高度受 5.5 英寸行高限制（超出会被裁剪）


**修改位置**：`scripts/blank_template.js` 中：
```js
const STRATEGY_ROW_HEIGHT_DXA = 7920;  // 5.5 英寸
// ...
analysisRow('教学策略', d.strategy, {
  img: resolveImage(d.strategyImg, 'strategy'),
  rowHeight: STRATEGY_ROW_HEIGHT_DXA,
  heightRule: 'exact',   // HeightRule.EXACT
}),
```

`analysisRow` 函数扩展支持 `opts.rowHeight`（DXA）+ `opts.heightRule`（`'exact'` / `'atLeast'`）参数。

### 教学流程图（v2.0 AI 生图 + 动态任务）

教学流程图是教案"教学分析"区的可视化核心，将教学设计转化为分层流程图。**v2.0 核心改进：任务数量不再固定为7个，而是根据教学内容动态构建（3~9个任务），阶段分配也按内容自动推断**。

#### 流程图规范

- **7 层 × N 列动态网格布局**（N = 任务数量，3~9）：
  - 第 1 层：标题栏（深蓝 #1F3A68 填充，白字）
  - 第 2 层：阶段划分（课前探究-橙 #E0873E / 课中实践-蓝 #2E6FB7 / 课后拓展-橙 #E0873E），按实际阶段动态分组
  - 第 3 层：N 个任务节点（蓝色 #2E6FB7 填充，编号 1~N）
  - 第 4 层：教师活动（绿色 #5BA85B 竖排标签 + 浅蓝 #E0EAF7 填充 + 棕 #9A7B4F 边框）
  - 第 5 层：学生活动（同上）
  - 第 6 层：学习诊断（白底 #FFFFFF + 绿 #5BA85B 边框 + 深绿 #244E24 文字）
  - 第 7 层：达标提级（浅黄 #FFE89A 填充 + 棕 #9A7B4F 边框 + 深棕 #6B4A0E 文字，加粗）
- **线条规范**：蓝色实线（跨层）+ 绿色虚线（教师→学生→诊断）+ 棕色虚线（诊断→达标），全部横平竖直
- **动态宽高比**：
  - ≤4 列：3:4（纵向）
  - 5~6 列：3:4（纵向）
  - 7 列：10:11（纵向）
  - ≥8 列：16:9（横纵均可）
- **标注语言控制**（`label_lang`）：
  - `"zh"`（默认）：课前探究/课中实践/课后拓展、教师活动/学生活动/学习诊断/达标提级
  - `"en"`：Pre-class Inquiry/In-class Practice/Post-class Extension、Teacher/Student/Diagnosis/Target

#### 动态任务提取策略

1. **优先从 flowChart 文字描述提取**：按编号标记（T1/T2/... 或 步骤1/...）或阶段关键词分割，动态识别任务数量
2. **LESSON_DATA 教学过程表补充**：从 `startPart/preparePart/basicPart1~3/endPart/postClass` 等字段补充
3. **模板自动选择兜底**：当两者都不足时，`_auto_detect_template()` 根据内容特征从9种预设模板中选择：
   - `basic_3`（3任务：准备+基本+结束）
   - `basic_4`（4任务：导入+准备+基本+结束）
   - `basic_5`（5任务：导入+准备+基本×2+结束）
   - `compact_5`（5任务：准备+基本×3+结束）
   - `deep_5`（5任务：导入+准备+基本×2+放松）
   - `standard_6`（6任务：课前+导入+准备+基本×2+结束）
   - `full_7`（7任务：课前+导入+准备+基本×2+结束+课后）
   - `extended_8`（8任务：课前+导入+准备+基本×3+结束+课后）
   - `full_9`（9任务：课前+导入+准备+基本×4+结束+课后）

### 教学反思

教学反思是教案的末栏，决定了整份教案的学术质量。

**重要：单元格内不重复标题**

教案表格中"教学反思"行的左侧标签已显示"教学反思"三字，因此反思文本**不要以 `# 教学反思` 开头**（避免 Word 单元格中重复出现两次标题）。直接从二级标题开始：

```
## 成效与亮点
...

## 问题与归因
...

## 改进方向
...
```

#### 三段式结构

反思必须包含三个二级标题，缺一不可：

| 二级标题 | 字数 | 内容要求 |
|---------|------|---------|
| **成效与亮点** | 80-100 字 | 本节课最成功的一个教学决策或瞬间，附学生表现/数据证据 |
| **问题与归因** | 80-100 字 | 本节课最突出的一个问题，结合理论/学情/设计层面归因（不只描述现象） |
| **改进方向** | 40-60 字 | 下次课/同类型课的具体、可操作的改进动作 |

#### 质量要求

- 总字数严格控制在 200-300 字之间（±20 字）
- 拒绝空话套话（禁止"学生积极性高""教学效果良好"等无内容表述）
- 所有判断必须有具体证据（学生行为、对话、数据）
- 归因要触及理论根源（运动技能形成规律、自我决定理论、分层教学原理等）
- 改进方向要具体到下节课可执行的动作

#### 反思文本示例（注意：不要 `# 教学反思` 标题）

```markdown
## 成效与亮点

本节课采用分层教学法处理学生运球基础差异问题，收到明显效果。将学生按基础分成 A/B/C 三组后，A 组学生在 25 分钟内独立完成行进间运球上篮动作的成功率从课初的 32% 提升至 71%，且 C 组学生因获得适配难度的练习而保持较高投入度。

## 问题与归因

基本部分中"分组对抗赛"环节出现动机分化，约 15% 的女生因竞争压力选择消极参与。从自我决定理论视角看，原因在于该环节过度强调胜负结果，削弱了能力感较弱学生的自主性动机。

## 改进方向

下次课将该环节改为"合作挑战赛"，按异质分组并设置团队共同目标，使每位学生的参与都能贡献团队得分。
```
