# 体育课教案生成技能

## 快速参考（TL;DR）

**功能**：根据用户提供的教学信息生成标准体育课教案 `.docx` 文件（单一表格、9 列网格）。

**核心原则：教案内容必须基于人才培养方案、教学大纲、教学标准、教学计划、教学设计逐层推导，各环节与之紧密关联。所有图片提示词必须从教案实际内容提取参数，禁止凭空捏造。**

### 三种模式

| 模式 | 触发场景 | 调度文档 |
|------|---------|---------|
| **从零生成** | 用户直接提供教案信息（或上传上游文档） | [04-workflow.md](docs/04-workflow.md) → 5阶段15步 |
| **完善教案** | 用户上传已有教案或提供部分信息 | [04-workflow.md](docs/04-workflow.md) → 完善教案 8 步 |
| **批量生成** | 用户上传授课计划/教学进度表 | [04-workflow.md](docs/04-workflow.md) → 批量生成 |

### 模式选择决策树

```
用户触发「教案」相关任务
├─ 有完整教案信息或上游文档？ ─→ 生成教案模式（5阶段15步）
│   阶段A: 上游文档链解析（培养方案→大纲→授课计划→推导）
│   阶段B: 教案内容构建 + 偏好确认
│   阶段C: 自动生成（策略/流程图/反思）
│   阶段D: 图片素材
│   阶段E: 预览确认 + 渲染交付
├─ 有部分信息 / 上传文件？ ───→ 完善教案模式（8步）
└─ 有授课计划 / 进度表？ ────→ 批量生成模式（授课计划为核心）
    └─ 提取结构化数据 → 逐周构建 → 校验 → 生成 → 确认 → 下一周
```

### 关键配置速查

| 参数 | 取值 | 说明 |
|------|------|------|
| `colorTheme` | `default` / `ocean` / `forest`（见 01-data-schema.md） | 表格配色，步骤 6 询问用户 |
| 队形图方式 | AI 生图(推荐) / PIL / 跳过 | 步骤 4 询问用户 |
| `label_lang` | `zh`(默认) / `en` | 图片标注语言 |
| 策略变量数 | 7 个（含课程进度上下文） | 步骤 5 收集 |
| 策略文本字数 | 200-300 字 | 步骤 5 生成 |
| 课时默认值 | 90 分钟（用户未指定时） | hours 字段 |
| JSON 禁用字符 | 中文顿号「、」 → 用分号替代 | 构建前必须检查 |

### 边界条件速查

> 关键边界处理规则摘要，完整列表见 [07-boundary-appendix.md](docs/07-boundary-appendix.md)。

| 场景 | 处理方式 | 文档引用 |
|------|---------|---------|
| 用户未提供上游文档 | 跳过阶段A，基于用户直接提供的信息构建 LESSON_DATA | 04-workflow.md |
| 用户未指定课时 | 默认 90 分钟 | 01-data-schema.md |
| 校徽文件缺失 | 自动切换为纯文字页眉（半透明浅灰底+深灰中文+中灰英文） | blank_template.js |
| AI 生图失败 | 策略图→PIL fallback→占位符；队形图→PIL→占位符 | 02-image-spec.md |
| JSON 含中文顿号「、」 | 构建前替换为分号，否则解析报错 | 01-data-schema.md |
| 批量生成数据偏差 | 5 种偏差模式自动校验 + 人工确认 | 07-boundary-appendix.md |

### 用户确认检查点

> Agent 在以下节点**必须暂停等待用户确认**，不得跳过或自主决策。

| # | 检查点 | 时机 | 确认内容 | 默认值 |
|:-:|--------|------|---------|--------|
| C1 | **模式确认** | 解析用户意图后 | 确认进入哪种模式（生成/完善/批量） | 自动判断 |
| C2 | **偏好确认** | 阶段B构建内容后、生成图片前 | 配色方案 + 队形图方式 + 标注语言 | default/AI/zh |
| C3 | **内容校验** | LESSON_DATA JSON 构建完成后 | 展示关键字段（课题/目标/重难点/课时），用户确认无误后再生成 | — |
| C4 | **预览确认** | docx 渲染完成后 | 展示最终文件，用户确认满意后交付；不满意则修改对应环节 | — |

**注意**：C3（内容校验）是 v5.2.0 新增的关键检查点——在投入大量图片生成之前，确保教案核心内容正确。

### 图片生成策略一览

| 图片类型 | 生成方式 | 数据源 |
|---------|---------|--------|
| **教学策略图** | **仅 AI 生图** (2:3) | `strategy` 策略文本 + sport/topic/target/课程进度 |
| **教学流程图** | **仅 AI 生图** (动态比例) | `flowChart` 流程图文字 + LESSON_DATA 教学过程；提取失败留空 |
| **课前图** | **仅 AI 生图** | `preClass` 课前活动描述 |
| **动作图** | **仅 AI 生图**（每环节独立） | 各 `basic[].content` 具体教学内容；调用 `get_action_ai_prompt_from_content()` |
| **预计生理负荷图** | **仅 AI 生图**（或 PIL 备选） | `exerciseDensity`/`avgHeartRate`/教学过程时间强度 |
| 队形图 | AI 生图(优先) → PIL → 占位符 | `sport`/`phase`/`org_method`/`n_students` |

> 完整图片规范（尺寸/DXA/风格/fallback）见 [02-image-spec.md](docs/02-image-spec.md)

### 核心脚本速查

| 脚本 | 功能 | 关键接口 |
|------|------|---------|
| `blank_template.js` | 渲染 .docx | `LESSON_DATA` 环境变量 |
| `strategy_prompt.md` + `strategy_ai_generator.py` | 策略文本 + 策略图 | 7 变量, `curriculum_context`, `label_lang` |
| `flowchart_ai_generator.py` | 流程图 AI 提示词 | `flowchart_text`, `lesson_data`, `label_lang` |
| `formation_ai_generator.py` | 队形图 AI 提示词 | `label_lang`, `sport`, `phase` |
| `action_image_generator.py` | 动作图 AI 提示词 (v4.8.0) | `get_action_ai_prompt_from_content(sport, action_content)` |
| `heart_rate_curve_v2.py` | 心率曲线 PIL | `data_points`, `total_minutes` |
| `reflection_prompt.md` | 教学反思规范 | — |

### 渲染交付速查

| 项目 | 规范 | 说明 |
|------|------|------|
| 输出格式 | `.docx` | blank_template.js 渲染 |
| 渲染命令 | `LESSON_DATA='{"..."}' node blank_template.js > output.docx` | JSON 通过环境变量传入 |
| 校徽 | `assets/school_logo.png`（默认自动检测） | 有校徽→双列叠加；无校徽→纯文字叠加 |
| 输出路径 | 工作目录（`/home/lingxi/workspace`） | 文件名：`第X课_课题名称.docx` |
| 页眉 | 半透明浅灰底(#E0E0E0) + 深灰中文(#444444) + 中灰英文(#999999) 叠加 | 自动生成，无需用户配置 |

## 子文档索引（按执行顺序逐步加载）

> **逐步披露原则**：Agent 按执行阶段按需加载对应文档，不要一次性全部读取。
> 每个 `→` 表示该步骤需要先读取箭头指向的文档。

| 优先级 | 文档 | 内容 | 加载时机 |
|:------:|------|------|---------|
| **P0** | [01-data-schema.md](docs/01-data-schema.md) | LESSON_DATA JSON schema（配色/基本信息/教学分析/教学过程/评价反思） | 步骤 6 构建 JSON 前**必须读取** |
| **P1** | [02-image-spec.md](docs/02-image-spec.md) | 图片尺寸/DXA/风格/占位符/生成策略 | 步骤 5-9 生成图片前读取 |
| **P1** | [03-auto-generation.md](docs/03-auto-generation.md) | 教学策略/流程图/反思自动生成逻辑 | 步骤 5-7 自动生成教学分析区时**必须读取** |
| **P0** | [04-workflow.md](docs/04-workflow.md) | 交互流程（生成教案13步 / 完善教案8步 / 批量生成含防错） | 确认模式后**立即读取** |
| **P2** | [05-template-structure.md](docs/05-template-structure.md) | 模板表格布局/列宽 DXA/心率图 V2 | 步骤 11 渲染 .docx 时读取 |
| **P2** | [06-formation-action.md](docs/06-formation-action.md) | 队形图 + 动作图详解（AI/PIL/策略/工作流） | 步骤 8-9 生成队形图/动作图时读取 |

### 典型加载路径

```
生成教案模式：
  SKILL.md → 04-workflow.md（阶段A步骤1-4: 上游文档链解析，无需子文档）
           → 01-data-schema.md（阶段B步骤5: 补全字段）
           → 03-auto-generation.md（阶段C步骤7-10: 策略/流程图/反思）
           → 02-image-spec.md + 06-formation-action.md（阶段D步骤11: 图片）
           → 05-template-structure.md（阶段E步骤14: 渲染）

完善教案模式（8步）：
  SKILL.md → 04-workflow.md（仅"完善教案"部分）→ 01-data-schema.md
           → 03-auto-generation.md → 02-image-spec.md

批量生成模式：
  SKILL.md → 04-workflow.md（仅"批量生成"部分）→ 01-data-schema.md
           → 03-auto-generation.md → 02-image-spec.md
```
